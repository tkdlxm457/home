 2. scope는 전역스코프와 지역스코프가 있다.

//아래 코드를 실행하면 111이 출력된다 그 이유는 a를 출력하기 위해 자신의 지역 스코프에서 a가 있는지 찾아보고 있으면 111을 출력한다.
var a = 1; // 전역 스코프
function print() { // 지역 스코프
 var a = 111;
 console.log(a);
}
print(); // 111
console.log(a); // 1

//아래 예제를 실행하면 1이 출력된다. 그 이유는 지역 스코프에 a가 존재하지 않기 때문에 지역에서 빠져나와 변수를 찾고 출력하기 때문이다. 
var a = 1; // 전역 스코프
function print() { // 지역 스코프
 console.log(a);
}
print(); // 1

3. var, let, const

var: 변수 선언 전 사용해도 에러x, 이미 선언되어있는 이름과 같은 이름으로 변수를 또 선언해도 에러x, 변수 선언시 초기값을 주지 않아도 됨, 값을 다시 할당할 수 있음
let: 변수 선언 전 사용하면 에러발생, 이미 존재하는 변수와 같은 이름으로 변수를 선언할 수 없음, 변수 선언시 초기값을 주지 않아도 됨, 값을 다시 할당할 수 있음
const: 변수 선언 전 사용하면 에러발생, 이미 존재하는 변수와 같은 이름의 변수를 선언할 수 없음, 반드시 초기값을 할당해야 함, 한번 할당한 값은 변경할 수 없음
var는 버그 발생과 메모리 누수의 위험등이 있기 때문에 var말고 let, const를 사용하는 것이 좋다.

4. 인터프리터 언어

인터프리터란 소스코트를 바로 실행하는 컴퓨터 프로그램 또는 환경을 말한다. 인터프리터는 소스코드를 한줄 한줄 읽어들이면서 실행하는 프로그램이다. 인터프리터는 번역과 실행이 동시에 이루어진다. 즉 별도의 실행파일이 존재하지 않는다.
